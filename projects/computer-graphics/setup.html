Creating The Game Loop

In SDL, one first has to call the init method to generate the underlying subsystems that will be used.

After things were initialized the next step was to create a window and a surface.

The general idea is that a surface is a buffer that you can draw to, and a window is the area of the screen that you can display the contents of a surface.

The next important step is to create a loop that holds the program open so that the window doesn't close the instant it is created. To do this, I created a loop that checks if the user wants to quit the application. Then I created a loop that handels events. If the quit event occurs, then it will set the quit flag to true on the outer loop and allow the program to close.

After exiting the loop some cleanup needs to be done to exit the program nicely.

At this point I had a program with an openable and closeable window. Perfect!

Limiting The Frame Rate

At the moment my program opens a blank window. It also uses a lot of resources, because there are no mechanisms to limit how fast the computer will execute the program. This is inefficient. To make my computer utilize fewer resources, I decided to cap the frame rate by adding a delay.

This is done by calculating how long it takes to get through the game loop. Then that time is used to figure out how much delay is needed to get the desired frame rate.

SDL gives back the time in milliseconds, so to achieve a desired frame rate one can use the formula " 1000 / frame_rate " to get how much time should pass each time you enter the game loop.

This can be calculated by getting the time when entering the loop (start_time) and the time when leaving the loop (end_time). Then you subtract the starting time from the ending time to get the total amount of time that has elapsed going through the loop (delta_time). Then we can sue the formula " (1000 / frame_rate) - delta " to get the amount of time that we need to delay to achieve our desired frame rate.

Recap

If we want a frame rate of 60 frames per second, then we know that it should take " 1000 / 60 " milliseconds to go through the loop.

frame_rate = 60fps
desired_delta = 1000 / 60

The time it actually took to get through the loop is given by subtracting the ending time from the start time

actual_delta = time_end - time_start

Finally, we wait so that the time elapsed is equal to our desired delta time, and how long to wait is given by the following formula:

delay = desired_delta - actual_delta

Drawing A Pixel to The Screen

Now that I have a window, a suface to draw to, and am limiting the frame rate the only thing left before I can go through the book is to draw a pixel to the screen.

A Renderer is needed to draw things in SDL. If you want to draw to a window use a renderer, but to draw to a surface use a softwareRenderer.

Drawing is a two step process:
1. set the drawing color
2. draw something to the screen

To set hte drawing color call the SetRenderColor fuction and pass it the renderer to set its color.

To draw a pixel use the RenderPlacePoint function to draw a point at an (x,y) coordinate on the surface.
